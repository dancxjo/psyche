## 2024-05-18 - ROS 2 ObjectDetector Processing Bypass
**Learning:** Found a major codebase-specific performance pattern in ROS 2 node callbacks. The `image_callback` was processing incoming camera frames (converting cv2 formats, detecting objects with morphological operations, finding contours) regardless of downstream consumers. In ROS 2, `get_subscription_count()` on publishers can be checked to cleanly skip expensive high-frequency operations when no one is listening. Additionally, constant objects like Numpy kernels in high-frequency callbacks cause unnecessary memory allocations each frame.
**Action:** When working on ROS 2 components with expensive sensor processing or high-frequency callbacks, always add an early return using `publisher.get_subscription_count() == 0` for all output topics to bypass the callback. Pre-allocate constant items (like masks, kernels) in `__init__`.
